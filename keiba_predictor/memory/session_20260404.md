## 2026-04-03〜04 修正まとめ

### エンコーディング問題（根本原因）
- nar.netkeiba.com は全ページ EUC-JP だが Content-Type で charset= が空を返す
- requests の apparent_encoding が iso-8859-5（キリル文字）等を誤検出する
- BS4 の from_encoding が無視されるケースがある
- **解決**: resp.content.decode(detected) で明示デコードしてからBS4に渡す方式に統一

### 払戻金パース問題
- NAR の払戻テーブルは1行に複数エントリが連結される
- 馬連: combo=['6','8'] amt=['370円'] → 2:1不一致でフォールバック失敗
- ワイド: combo 6個 amt 3個 → グループ化が必要
- **解決**: combo数÷amt数で等分割するグループ化ロジックを追加

### 出馬表テーブルセレクタ問題
- table.Shutuba_Table が PredictRap_Table（2行のみ）にヒット
- **解決**: table.ShutubaTable:not(.PredictRap_Table) を優先

### GitHub Actions
- permissions: contents: write が全ワークフローに必要
- cron スケジュールが動かない → cron-job.org で外部トリガーに切替
- db.netkeiba.com の tbody が空の場合 race.netkeiba.com にフォールバック

**Why:** NAR版を新規構築する際、JRA版のコードをベースにしたが、NARサイト固有の仕様（EUC-JP、テーブル構造、払戻形式）への対応が不十分だった。
**How to apply:** NAR関連のスクレイピングでは常にencoding="euc-jp"を明示し、テーブルパースはNAR固有の構造に対応させる。
